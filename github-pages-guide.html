GitHub Pages 完全ガイド

無料でWebサイトを公開する最速の方法
01 GitHub Pages とは？
GitHubリポジトリ = Webサイト

リポジトリにHTMLファイルを置いてpushするだけで
世界中に公開されるWebサイトになる、無料のホスティング機能

💻
ローカル
HTMLを編集
→
📦
git push
GitHubに送信
→
🌐
公開!
username.github.io

できること

  • HTML / CSS / JavaScript の静的サイト
  • ポートフォリオ、ブログ、ドキュメント
  • 独自ドメインの設定
  • HTTPS 自動対応

できないこと

  • サーバーサイド処理（PHP, Python等）
  • データベース接続
  • ユーザー認証（サーバー側）
  • 動的API
02 始め方（3ステップ）
1
GitHubにリポジトリを作成

公開用のリポジトリを作る。Public でも Private でもOK。

↓
2
HTMLファイルをpush

index.html をルートに置いてpushする。これがトップページになる。

↓
3
GitHub Pages を有効化

Settings > Pages で公開ブランチを選択。数分後にURLが発行される。

実際のコマンド

# 1. リポジトリ作成 & push git init git add index.html git commit -m "initial commit" gh repo create my-site --public --source=. --push # 2. GitHub Pages を有効化（CLIから） gh api repos/{owner}/{repo}/pages -X POST \ -f source[branch]=main -f source[path]=/ # 3. 公開URL https://{username}.github.io/{repo-name}/
GUI派の場合: GitHub.com でリポジトリを開き、Settings > Pages > Source で「Deploy from a branch」を選び、main / root を指定するだけ。
03 URL の仕組み
種類URL形式説明
ユーザーサイト username.github.io リポジトリ名をusername.github.ioにすると、ルートURLになる。1アカウント1つだけ。
プロジェクトサイト username.github.io/repo 通常のリポジトリ。URLはリポジトリ名がパスになる。いくつでも作れる。
独自ドメイン www.example.com DNS設定で独自ドメインを紐付け可能。HTTPS も自動で対応。
例: 今回の診断ページ
# リポジトリ github.com/sol-111/entj-intj-diagnosis ~~~~~~~~ ~~~~~~~~~~~~~~~~~~~~~~ ユーザー名 リポジトリ名 # 公開URL（自動生成） https://sol-111.github.io/entj-intj-diagnosis/
04 ファイル構成のルール
# 基本構成 my-repo/ ├── index.html ← トップページ（必須） ├── about.html ← /about.html でアクセス ├── css/ │ └── style.css ← /css/style.css ├── js/ │ └── main.js ← /js/main.js └── images/ └── logo.png ← /images/logo.png
ルール内容
トップページindex.html がルートURLに対応。必ずルート直下に置く。
パス = URLファイルの配置がそのままURLのパスになる。
大文字小文字URLは大文字小文字を区別する。ファイル名は小文字推奨。
404ページ404.html をルートに置くとカスタムエラーページになる。
05 デプロイ方法の比較
方式仕組みおすすめ度
Deploy from branch
簡単		 指定ブランチの内容をそのまま公開。pushするだけで自動反映。今回使っている方式。 ★★★★★
GitHub Actions
中級		 ビルド処理（React, Next.js 等）を挟んでからデプロイ。フレームワーク利用時に必要。 ★★★★☆
今の構成（HTML直置き）なら「Deploy from branch」で十分。フレームワークを使い始めたらGitHub Actionsに切り替える。
06 更新の流れ
1
ローカルでHTMLを編集

好きなエディタで index.html を修正。ブラウザで確認。

↓
2
コミット & プッシュ

git add . && git commit -m "update" && git push

↓
3
自動デプロイ（1〜2分）

pushしたら GitHub が自動でサイトを更新。何もしなくてOK。

デプロイ状態の確認

# デプロイ状況をCLIで確認 gh api repos/{owner}/{repo}/pages --jq '.status' # "built" と出れば公開済み
07 制限と注意点
項目制限
リポジトリサイズ推奨 1GB 以下 注意
サイトサイズ公開サイトは 1GB 以下
帯域制限月間 100GB のソフトリミット
ビルド回数1時間に 10回 まで
禁止用途ECサイト、SaaS、商用ホスティング等は規約違反 注意
個人プロジェクトやポートフォリオなら全く問題なし。アクセスが増えてきたら Vercel や Netlify への移行を検討。
08 他のホスティングとの比較
サービス料金難易度特徴
GitHub Pages 無料 簡単 git pushだけ。静的サイト専用。
Vercel 無料枠あり 簡単 Next.js / React に最適。サーバーレス関数も使える。
Netlify 無料枠あり 簡単 フォーム機能あり。ビルド設定が柔軟。
Firebase Hosting 無料枠あり 普通 Google製。認証・DBと組み合わせやすい。
レンタルサーバー 月額有料 普通 PHP等サーバーサイドも動く。自由度高い。
09 よく使うコマンドまとめ
# リポジトリ作成 & Pages有効化 gh repo create my-site --public --source=. --push gh api repos/{owner}/my-site/pages -X POST \ -f source[branch]=main -f source[path]=/ # 更新してデプロイ git add index.html git commit -m "update" git push # デプロイ状態を確認 gh api repos/{owner}/my-site/pages --jq '.status' # Pages情報を見る gh api repos/{owner}/my-site/pages --jq '.html_url' # 独自ドメインを設定 gh api repos/{owner}/my-site/pages -X PUT \ -f cname=www.example.com
